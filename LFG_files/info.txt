Files that must be added/modified in the GENIE code in order
to implement the LFG model:

config/LFGNuclearModel.xml
src/Nuclear/LFGNuclearModel.h
src/Nuclear/LFGNuclearModel.cxx

Modified:

Summary:

config/LFGNuclearModel.xml
src/Nuclear/LFGNuclearModel.h
src/Nuclear/LFGNuclearModel.cxx
These files actually implement the local Fermi gas model. It calculates the Fermi momentum using a local Fermi gas, then generates a momentum for the nucleon below the Fermi momentum.

config/master_config.xml
One line added to link to LFGNuclearModel.xml.

config/ Messenger files updated to include LFGNuclearModel
Use the same messenger options as BodekRitchie for LFGNuclearModel.

config/UserPhysicsOptions.xml
Add LFGNuclearModel as an option.

src/EVGModules/FermiMover.cxx
Call GenerateNucleon() with a target and a radius, so the LFGNuclearModel class has access to the radius.

src/EVGModules/PauliBlocker.h
src/EVGModules/PauliBlocker.cxx
Updated to check what type of nuclear model is being used, and use a local Fermi gas for pauli blocking if the nuclear model is LFG.

src/Nuclear/NuclearModel.h
One line added to include kNucModelLocalFermiGas as an option.

src/Nuclear/LinkDef.h
One line added to link the LFGNuclearModel class to the GENIE code.

src/Nuclear/NuclearModelI.h
Added the option for GenerateNucleon() to be called with a radius. The default is to call the GenerateNucleon() method that does not ask for a radius.

src/Nuclear/NuclearModelMap.h
src/Nuclear/NuclearModelMap.cxx
Updated to allow GenerateNucleon() to be called with a radius. FermiMover calls NuclearModelMap::GenerateNucleon(), which in turn calls LFGNuclearModel::GenerateNucleon().
